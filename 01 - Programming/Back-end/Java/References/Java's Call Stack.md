# Call Stack

## How Java's Call Stack Manages Execution
هر برنامه‌ی جاوا برای پیگیری اینکه کدام method درحال اجراست و کدام متد پس از ان متد باید اجرا شود به **Call Stack** نیاز دارد. یک LIFO data structure است هروقت یک متد `call` می‌شود جاوا یک Stack frame را روی `call stack` پوش می‌کنه هروقت متد تموم‌بشه frameش حذف می‌شود. 
شروع هر برنامه جاوا از متد Main ان است. (entry point of execution) هروقت برنامه شروع بشه هر متدی که `call`‌بشه یک frame جدید به `stack` اضافه می‌شود اگه متد یک متد دیگری رو صدا بزنه جاوا یک `new frame` دیگه روی متد قبلی اضافه می‌کنه.

یک stack frame شامل چندین قطعه از information میشه که برای اجرای برنامه نیاز است (Local var, parameters, return address) -  return address به جاوا می‌گوید اجرای را پس از تکمیل متد از کجا از سربگیرد. بعضی از متد ها ممکنه value های موقت رو در stack frame خودشون داشته باشند



**نکته:** وقتی یک متد با `return` تمام می‌شود، **قبل از اینکه Stack Frame آن از Call Stack حذف (Pop) شود، مقدار `return` به متد فراخواننده (Caller) منتقل می‌شود.** سپس Frame متد حذف شده و اجرای برنامه از دستور بعد از فراخوانی آن متد ادامه پیدا می‌کند. به همین دلیل، با وجود حذف شدن Frame متد، مقدار بازگشتی همچنان در دسترس متد فراخواننده است.
